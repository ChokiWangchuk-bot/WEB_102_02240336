Social Media API Documentation

RESTful API for a social media platform. Base URL: http://localhost:3000/api

Auth: Protected routes require header X-User-Id: {userId}

Users

GET/api/users

Get a paginated list of all users.

Query Parameters

ParameterTypeDescription
pageNumberPage number (default: 1)
limitNumberNumber of users per page (default: 10)

Response (200)

{ "success": true, "count": 3, "page": 1, "total_pages": 1, "data": [...] }
GET/api/users/:id

Get a specific user by ID.

Response (200)

{ "success": true, "data": { "id": "1", "username": "traveler", ... } }
POST/api/users

Create a new user.

Request Body

{ "username": "new_user", "email": "new@example.com", "password": "secret", "full_name": "New User", "bio": "Hello!" }

Response (201)

{ "success": true, "data": { "id": "4", "username": "new_user", ... } }
PUT/api/users/:id

Update an existing user. (Requires X-User-Id header)

DELETE/api/users/:id

Delete a user. (Requires X-User-Id header)

Response (200)

{ "success": true, "data": {} }

Posts

GET/api/posts

Get all posts with user info.

GET/api/posts/:id

Get a specific post by ID.

POST/api/posts

Create a new post. (Requires X-User-Id header)

Request Body

{ "caption": "My post!", "image": "https://example.com/image.jpg" }
PUT/api/posts/:id

Update a post. (Requires X-User-Id header — must be post owner)

DELETE/api/posts/:id

Delete a post. (Requires X-User-Id header — must be post owner)

Comments

GET/api/comments

Get all comments.

POST/api/comments

Add a comment. (Requires X-User-Id header)

Request Body

{ "text": "Great post!", "post_id": "1" }

Likes

GET/api/likes

Get all likes.

POST/api/likes

Like a post. (Requires X-User-Id header)

Request Body

{ "post_id": "1" }
DELETE/api/likes/:id

Remove a like. (Requires X-User-Id header)

Followers

GET/api/followers/:userId/followers

Get all followers of a user.

GET/api/followers/:userId/following

Get all users that a user is following.

POST/api/followers/:userId/follow

Follow a user. (Requires X-User-Id header)

DELETE/api/followers/:userId/unfollow

Unfollow a user. (Requires X-User-Id header)
